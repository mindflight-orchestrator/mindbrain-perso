# OWL2 Import Integration Plan

Date: 2026-05-20
Scope: `mindbrain` standalone/SQLite codebase.

## Summary

MindBrain already has the storage pieces needed to carry ontology-like data:
ontology headers, namespaces, dimensions, values, raw entities, raw relations,
facet assignments, and graph rebuild paths.

The first integration now treats OWL2 as normalized RDF/N-Triples input,
imports useful vocabulary and assertions into existing raw tables, preserves
all source triples in `ontology_triples_raw`, and leaves full reasoning to a
later layer.

## Implementation Status

Implemented in the standalone SQLite runtime:

- `mindbrain-standalone-tool ontology-import --input <file.nt>` imports
  N-Triples into `ontology_triples_raw`;
- `--materialize-graph` mirrors IRI object triples into graph raw rows;
- `mindbrain-standalone-tool ontology-export --format ntriples` exports the
  preserved triples;
- `ontology-export --format bundle` emits a workspace taxonomies bundle;
- bundle import/export round-trips ontology triples and projected ontology rows;
- tests import all local `docs/source/*.nt` fixtures into SQLite and verify
  preservation/projection behavior.

Still out of scope: full OWL2 reasoning, remote ontology fetching, and native
parsers for RDF/XML, Turtle, Manchester Syntax, or OWL Functional Syntax.

## Current State

- `sql/sqlite_mindbrain--1.0.0.sql` defines ontology and raw graph tables:
  `ontologies`, `ontology_namespaces`, `ontology_dimensions`,
  `ontology_values`, `ontology_entity_types`, `ontology_edge_types`,
  `ontology_entities_raw`, `ontology_relations_raw`, `entities_raw`,
  `relations_raw`, and `facet_assignments_raw`.
- `src/standalone/collections_sqlite.zig` exposes bundle-style ontology
  loading helpers and typed specs for namespaces, dimensions, values, entity
  types, edge types, seed entities, and seed relations.
- `src/standalone/import_pipeline.zig` exposes the practical write API:
  ontology registration, collection attachment, facet assignment, entity
  upsert, relation insert, relation properties, document links, and reindexing.
- `src/benchmark/yago_import.zig` is useful as a reference for RDF-like input,
  but it is a benchmark importer, not a general OWL2/Turtle parser.
- `src/standalone/collections_io.zig` exports/imports collection bundles and
  now round-trips ontology body tables plus `ontology_triples_raw`.

## Target Import Contract

Support a first public import path for normalized RDF:

- N-Triples is the canonical input format.
- Turtle is accepted only after normalization to simple triples.
- RDF/XML, Manchester Syntax, and OWL Functional Syntax remain upstream
  conversion responsibilities.
- Every imported entity keeps its original IRI in metadata.
- Unsupported triples and complex expressions must be preserved, not dropped.

The importer should expose a standalone command such as:

```sh
mindbrain-standalone-tool ontology-import \
  --db path/to/mindbrain.sqlite \
  --workspace-id ws1 \
  --ontology-id example:owl \
  --input ontology.nt \
  --materialize-graph
```

## Mapping Rules

- `owl:Class` and `rdfs:Class` become `ontology_entity_types`.
- `rdfs:subClassOf` becomes both hierarchy metadata and a typed graph relation
  when class nodes are materialized.
- `owl:ObjectProperty` becomes `ontology_edge_types`.
- `owl:DatatypeProperty` becomes `ontology_dimensions` when it represents a
  facetable scalar, or relation/property metadata when it qualifies an edge.
- Individuals become `entities_raw` rows with stable URI-derived IDs.
- Object property assertions become `relations_raw`.
- Literal assertions become facet assignments, typed relation properties, or
  metadata depending on the subject kind.
- `owl:equivalentClass`, `owl:equivalentProperty`, `owl:disjointWith`,
  `owl:sameAs`, `owl:differentFrom`, `owl:inverseOf`, and
  `rdfs:subPropertyOf` become reserved typed relations between ontology nodes.
- `owl:Restriction`, `owl:someValuesFrom`, `owl:allValuesFrom`,
  cardinalities, intersections, unions, and blank-node class expressions are
  stored as JSON definitions and raw triples. They are not inferred in the
  SQLite MVP.

## Required Changes

- Add a normalized triple parser/importer instead of extending the YAGO
  benchmark importer directly.
- Add a stable URI-to-ID strategy, based on deterministic hashing of workspace,
  ontology ID, and full IRI.
- Add a lossless raw triple preservation surface, either as a new table or as a
  structured payload attached to ontology metadata.
- Extend bundle export/import so ontology namespaces, dimensions, values,
  entity types, edge types, ontology seed entities, and ontology seed relations
  round-trip.
- Decide whether ontology nodes are copied to `entities_raw` during import or
  whether graph reindexing should learn how to replay `ontology_entities_raw`
  and `ontology_relations_raw`.

## Acceptance Tests

- Import a normalized fixture containing classes, subclass hierarchy, object
  properties, datatype properties, individuals, labels, aliases, inverse
  properties, and a restriction blank node.
- Verify ontology tables contain the expected vocabulary.
- Verify graph raw tables contain materialized class/property/individual nodes
  when graph export is requested.
- Verify unsupported triples are preserved.
- Verify bundle export followed by import recreates the ontology body.
- Verify `reindex_all` makes imported graph nodes and relations queryable.

## Out Of Scope

- Full OWL2 DL reasoning.
- Network import from remote ontology IRIs.
- RDF/XML, Manchester Syntax, or OWL Functional Syntax parsing inside the
  standalone binary.
- LLM-assisted mapping or classification in the SQLite MVP.
